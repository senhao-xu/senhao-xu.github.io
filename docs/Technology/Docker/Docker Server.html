PrivateBin ​

前言 ​

PrivateBin是一个极简、开源的在线pastebin，服务器对粘贴的数据一无所知。PrivateBin|Github.

使用docker部署 ​

PrivateBin|Docker Hub docker镜像地址，该Wiki中有详细的使用方法.

创建一个PrivateBin容器

bash
docker run -d --restart="always" \
  --read-only \
  --name private-bin \
  -p 8080:8080 \
  -v $PWD/privatebin-data:/srv/data \
  privatebin/nginx-fpm-alpine

创建完成后需要对宿主机挂载的文件夹进行授权,否则会出现【送出】功能粘贴失败的问题,chmod -R 777 privatebin-data 进行授权.

部署完成后访问http://ip:8080进行访问，此时PrivateBin还无法使用，由于Private使用浏览器https进行数据加密所以需要使用nginx进行反代。

OpenWRT ​

github 1、打开网卡混杂模式, enp1s0: 你的本地网卡，使用 ip a 查看

bash
sudo ip link set enp1s0 promisc on

2、创建名称为macvlan的虚拟网卡，网段与你本地的网段一致

bash
docker network create -d macvlan --subnet=192.168.0.0/24 --gateway=192.168.0.1 -o parent=enp1s0 openwrt

3、拉取镜像

bash
docker pull registry.cn-hangzhou.aliyuncs.com/zzsrv/openwrt:x86_64

4、创建容器并后台运行

bash
docker run --restart always --name openwrt -d --network macnet --privileged zzsrv/openwrt:x86_64 /sbin/init

5、进入容器内部环境

bash
docker exec -it openwrt bash

6、根据自己实际情况修改网络配置，修改完成后保存配置; 网关修改为本地网关,IP为你要访问openwrt的IP

bash
vi /etc/config/network

7、退出容器内部环境，在宿主机环境执行重启容器命令

bash
docker container restart openwrt
